## How I work

- **Small PRs; revert when I'm wrong.** Inktrail has a nine‑commit revert to prove it.
- **Problem first, numbers second, scope third.** Every PRD opens with the diagnosis — what's failing, by how much, why now — before the prescription.
- **Phased shipping with rationale.** Phase 1 → 1.5 → 2 isn't arbitrary; sequencing comes from where leverage compounds fastest. Foundational measurement before features that depend on signal.
- **AI features are eval‑gated.** Langfuse + LLM‑as‑judge on Hub AI, cost reconciliation loops on every LLM call, server‑side validation that round‑trips errors back to the model so it self‑corrects in one or two turns.
- **Release day is a data event.** Every release ships with a feedback capture and an issue log. Retros happen in days, not weeks, and feed the next monthly report.
